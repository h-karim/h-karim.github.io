Overview

The goal of this project is to provide BIDS-iEEG compatibility for LORIS as well as visualization for EEG and iEEG recordings for the electrophysiology module by using an already existing React library.

Deliverables

  • Install LORIS and understand codebase
  • Work on issues from different modules and issue pull requests
  • Add iEEG support to LORIS-MRI
  • Connect visualization library to LORIS

Timeline

May: First Contact

I was introduced to the team behind LORIS in may as well as install it on a virtual machine hosted at the McGill Center for Integrative Neuroscience in order to familiarize myself with the codebase. Due to the pandemic at the time, I could not meet the team in person. Nevertheless, attending several meetings allowed me to take on issues from the backlog.

June

In June I worked on several issues spanning different LORIS modules (with a focus on the electrophysiology browser) in order to further develop my understanding of the software stack used. LORIS was going through a release testing phase which I participated in by providing feedback on the results of the testing plans. I was also fortunate enough to attend the OHBM virtual brainhack conference where I got to listen to a talk about BIDS.

July

July was marked by switching my attention to LORIS-MRI, a sister repo to LORIS used to upload EEG files in BIDS format to its database. I focused on reading the code as well as documentation on EEG and iEEG-BIDS. This allowed me to implement changes required to adapt the upload pipeline to support iEEG-BIDS directories. By the end of July iEEg data was successfully uploaded and a pull request was opened.
